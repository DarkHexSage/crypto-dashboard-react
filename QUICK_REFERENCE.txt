========================================
WHY THIS IS BETTER THAN FLASK
========================================

Flask approach:
- Need Flask server
- Python dependencies
- Complex favicon routing (APPLICATION_ROOT issues!)
- Rebuild Docker image for favicon changes
- Backend always running

React approach:
- Just static files after build
- No backend needed
- Favicon works out of the box
- Deploy anywhere (S3, Netlify, GitHub Pages, nginx)
- 100x faster
- Smaller Docker image
